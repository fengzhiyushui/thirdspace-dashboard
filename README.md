# Thirdspace Dashboard

Thirdspace Dashboard 是一个 Obsidian 原生工作台插件，用来把个人知识库里的今日简报、实时热点、任务状态、Agent 日志、项目上传包和 Vault 区域状态集中展示在一个总控台里。
This is a native Obsidian dashboard plugin for a Thirdspace-style knowledge system. It turns Markdown files generated by your local tools or cloud agents into a visual dashboard inside Obsidian.

> 本仓库是可公开发布的脱敏版本，只包含插件代码，不包含个人 Vault 内容、同步配置、服务器地址、日志、密钥或私人笔记。
> This public release contains only the plugin code. It does not include personal notes, sync configuration, server addresses, logs, credentials, or private vault data.

## 功能 / Features

- Obsidian 原生 `ItemView` Dashboard。
- 暖黄、青绿、夜间三套内置主题，可在界面内切换。
- GitHub 风格活动热力图。
- 今日实时热点时间流。
- 任务、热点、Agent 日志、项目上传包状态卡片。
- Vault 区域概览卡片，每行最多三个区域。
- 毛玻璃卡片、柔和 hover、鼠标光斑和响应式布局。

## 安装 / Installation

1. 下载或克隆本仓库。
2. 将整个文件夹放到你的 Obsidian Vault 插件目录：

```text
.obsidian/plugins/thirdspace-dashboard
```

3. 在 Obsidian 设置中关闭安全模式，或允许第三方插件。
4. 启用 `Thirdspace Dashboard`。
5. 打开命令面板，执行 `Open Thirdspace Workbench`。

## 期望的 Vault 路径 / Expected Vault Paths

插件会尝试读取下面这些 Markdown 路径。如果路径不存在，界面仍然可以打开，只是对应区域会显示为空或默认状态。

```text
10-Daily/today-brief.md
60-Tasks/open-tasks.md
70-Radar/today-hotspots.md
20-Projects/code/status.md
40-Wiki/Projects/code.md
90-Agent/Hermes-Logs/
90-Agent/Project-Uploads/
95-System/vault-schema.md
```

## 适合的系统 / Intended Use

适合以 Obsidian 作为前端、后台 Agent 或脚本持续写入 Markdown 的个人知识库系统，例如：

- Obsidian + Syncthing
- Obsidian + Hermes
- Obsidian + VPS 定时任务
- Obsidian + 本地 AI / Skill 项目进度同步

It is designed for Markdown-first personal knowledge systems where Obsidian is the main interface and background agents compile, summarize, or index information into the vault.

## 隐私说明 / Privacy

这个插件本身不上传文件，不连接服务器，也不包含远程同步逻辑。它只读取当前 Obsidian Vault 中的 Markdown 文件并渲染界面。
The plugin does not upload files, connect to servers, or implement remote sync. It only reads Markdown files inside the current Obsidian vault and renders a dashboard view.
